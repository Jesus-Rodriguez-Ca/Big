Accessibility

Accessibility in the Web

Just a few decades ago, we used to search for information in books, newspapers, magazines, and dictionaries. Nowadays, we have the Web to search for all that information. The Web is a great resource to find all kinds of content such as business, cooking, health, and jobs. In order that everyone is able to access this content, the Web must be accessible to all. Web designers have a responsibility to provide accessible content.

There are some accessibility standards in web design that designers have to keep in mind. One of the points that designers have to cover when building a website is color contrast. According to the World Health Organization, globally, there are about 2.2 billion people that have a vision impairment or blindness. Designers should consider not using color contrast as their main method of conveying important information. The W3C advocates standards to apply to websites in order to cover vision impairment and color blindness.

Moreover, the use of proper markups on code content is an important practice while writing code. Browsers render HTML elements structurally and hierarchically. The use of these elements to give style to the website may result in wrong rendering by screen readers. Screen readers are important to people who can’t see and need to hear the content. Here is an example by Oyetobe Tobi Emmanuel of the use of alt=”Description” HTML attribute: < src =”image-location” alt=” this is where your image description goes”>

Undoubtedly, keyboard accessibility is one of the most important practices. There are many people who have motor disabilities and people that are unable to see that need accessibility when using the keyboard. Web designers have to write their code in the best way to give everyone attainability. The University of Washington talks about some techniques that designers can use: Avoid Overriding Browser’ Default Focus Indicator and Enhance Browsers’ Default Focus Indicator.

Accessibility in web design is an important practice when building websites. Color contrast, hierarchically structured, and keyboard accessibility are important standards. Every web designer and web developer should follow the standards of accessibility. By following these standards, the Web becomes more inclusive and beneficial to all. There are more of these standards rules to learn, but these are a good start.
